# Выбор платформ виртуализации

## Задача 2

### Высоконагруженная база данных MySQL

**Выбор:** Физические серверы.

**Причины:**

* максимальная производительность;
* минимальные задержки дисковой подсистемы;
* отсутствие влияния других виртуальных машин;
* высокая надёжность.

### Web-приложения

**Выбор:** Виртуализация уровня ОС (контейнеры).

**Причины:**

* быстрое развертывание;
* эффективное использование ресурсов;
* удобное масштабирование;
* высокая плотность размещения сервисов.

### Windows-системы бухгалтерии

**Выбор:** Паравиртуализация.

**Причины:**

* хорошая изоляция систем;
* удобное резервное копирование;
* простое администрирование;
* поддержка Windows-приложений.

### Высокопроизводительные расчёты на GPU

**Выбор:** Физические серверы.

**Причины:**

* максимальная производительность GPU;
* отсутствие накладных расходов виртуализации;
* лучшая совместимость с вычислительными библиотеками.

# Выбор системы управления виртуализацией

## Задача 3

### 100 виртуальных машин Linux и Windows, Windows-инфраструктура

**Выбор:** VMware vSphere, Hyper-V

**Причины:**

* удобное централизованное управление;
* поддержка балансировки нагрузки;
* репликация и резервное копирование;

### Бесплатное Open Source решение для 20–30 серверов

**Выбор:** KVM

**Причины:**

* бесплатное и открытое решение;
* высокая производительность;
* поддержка Linux и Windows;
* встроенные средства резервного копирования.

### Бесплатная виртуализация Windows-инфраструктуры

**Выбор:** Hyper-V.

**Причины:**

* высокая совместимость с Windows;
* хорошая производительность;
* интеграция с инфраструктурой Microsoft;

### Тестирование ПО на нескольких дистрибутивах Linux

**Выбор:** VirtualBox, KVM

**Причины:**

* простота использования;
* поддержка множества дистрибутивов;
* возможность создавать снимки состояния;
